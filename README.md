# ⚡ Bolt — Ansible Automation Platform

A self-hosted Ansible automation platform for learning host management,
playbook automation, and infrastructure tooling.

## Quick Start

```bash
# 1. Extract and enter the project
tar -xzf bolt.tar.gz
cd bolt

# 2. Run setup — it does everything for you
chmod +x setup.sh
./setup.sh          # interactive: asks how many hosts
./setup.sh 5        # or directly say how many hosts you want

# 3. Open the UI
http://localhost:8080
```

## What setup.sh does

1. Generates SSH key pair (private + public) in `ssh-keys/`
2. Writes `docker-compose.yml` with your chosen number of hosts
3. Writes `ansible/inventory/hosts.yml` with all host IPs
4. Runs `docker compose up --build` to start everything

## Commands

```bash
./setup.sh            # interactive setup
./setup.sh 3          # spin up 3 hosts directly
./setup.sh --destroy  # stop and remove everything
docker compose down   # just stop containers
```

## Adding a Playbook

1. Create `ansible/playbooks/MyPlaybook.yml`
2. Create `ansible/playbooks/MyPlaybook.meta.json`:

```json
{
  "description": "What this playbook does",
  "inputs": [
    { "name": "my_var",  "label": "My Variable", "type": "text",   "default": "value" },
    { "name": "version", "label": "Version",     "type": "select", "options": ["1.0","2.0"] }
  ]
}
```

## Project Structure

```
bolt/
├── setup.sh                  ← Run this first
├── docker-compose.yml        ← Auto-generated by setup.sh
├── frontend/                 ← Web UI (HTML/JS + Nginx)
├── backend/                  ← Flask API + Ansible engine
├── ansible/
│   ├── ansible.cfg
│   ├── inventory/hosts.yml   ← Auto-generated by setup.sh
│   └── playbooks/            ← Your playbooks go here
├── docker/
│   └── target-host/          ← SSH target container image
└── ssh-keys/                 ← Auto-generated by setup.sh
    ├── bolt_rsa              ← Private key (backend uses this)
    └── bolt_rsa.pub          ← Public key (baked into target hosts)
```
